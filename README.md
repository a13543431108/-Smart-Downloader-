Smart Downloader 智能多协议下载器
一个基于 Python + Tkinter 构建的图形化多协议下载管理工具，支持 HTTP/HTTPS、FTP、本地文件及 Blob 协议，采用多线程分块下载、断点续传、完整性校验等特性，并内置本地 HTTP 服务以接收浏览器扩展推送。

✨ 主要特性
  多协议支持：HTTP/HTTPS、FTP、file://、blob:、data: URI
  多线程分块下载：动态调整线程数（最大 100+），支持大文件高速下载
  断点续传：自动保存任务状态，下次启动可继续未完成的下载
  分块完整性校验：SHA-256 校验每个分块，确保数据完整（可配置）
  内存缓存加速：将下载数据先缓存至内存，达到阈值后写入磁盘，减少磁盘 I/O
  智能重定向解析：自动跟踪链接重定向，获取真实文件名
  预连接技术：在下载前预先建立连接池，提升响应速度
  自动线程调整：根据当前下载速度动态调整线程数，平衡性能与资源
  网络吞吐量监控：实时显示系统整体下载/上传速度
  代理支持：支持 HTTP/HTTPS/SOCKS5 代理
  浏览器扩展联动：启动本地 HTTP 服务（端口 9876），可接收浏览器扩展推送的下载链接
  丰富的设置界面：可配置默认目录、超时、重试次数、缓存大小、分块大小、主题等
  任务管理：支持暂停/继续、删除、重试失败任务、一键开始/停止所有任务

🚀 快速开始
  环境要求
  Python 3.6+
  Windows / macOS / Linux

  安装依赖（进入下载器(download)文件夹中）
    在cmd中输入
    pip install -r requirements.txt
  直接运行：
    python 下载器\(优化\).py
  使用说明
    在“下载”页输入 URL（自动识别协议），选择保存路径，点击“开始下载”。
    在“任务”页查看所有任务状态，可暂停/继续、删除或打开已完成文件。
    在“系统”页查看内存使用、网络速度及操作日志。
    在“设置”页自定义线程数、缓存大小、代理、分块校验等高级选项。

浏览器扩展联动
  介绍：
    在下载器程序启动后会在 http://127.0.0.1:9876 监听 POST 请求（路径 /add_task），支持第三方扩展通过 JSON 推送下载任务（示例格式见下文）。
    你可以在浏览器扩展中调用此接口，实现一键发送链接到下载器。注意：如果没有打开下载器，浏览器扩展不会起到作用，会使用会浏览器本身下载。
  用法：
    在edge浏览器中点击扩展，点击扩展管理，打开开发人员选项，点击“加载解压缩的扩展”，选择download with edge文件夹的位置，即可安装成功

🛠️ 技术架构
  GUI框架：Tkinter + ttk
  网络请求：requests（带连接池、重试机制）
  系统监控：psutil（内存、网络）
  并发控制：concurrent.futures.ThreadPoolExecutor
  数据持久化：resume_info.json（任务状态）、downloader_settings.json（用户配置）
  分块管理：每个任务独立临时目录（.{task_id}），存储分块文件和元数据（meta.json）

📦 扩展推送 API
  端点：POST http://127.0.0.1:9876/add_task
  请求体（JSON）：

json
{
  "url": "https://example.com/file.zip",
  "filename": "自定义文件名.zip",   // 可选
  "headers": {                    // 可选，自定义请求头
    "Cookie": "session=xxx"
  }
}
响应：{"status": "ok"}

📂 文件说明
  下载器(优化).py：主程序源码
  resume_info.json：断点续传信息（自动生成）
  downloader_settings.json：用户设置（自动生成）
  d.ico：程序图标（可选，需与主程序同目录）

❓ 常见问题
  Q: 下载失败提示“分块校验不匹配”怎么办？
  A: 检查网络稳定性，或在“设置”中禁用“分块完整性校验”后重试（不推荐，可能影响数据完整性）。

  Q: 如何修改默认下载目录？
  A: 在“设置”页的“默认下载目录”中浏览选择，点击“保存设置”即可。

  Q: 为什么网络速度显示为 0？
  A: 请确保以管理员权限运行（Windows）或具有网络监控权限（Linux/macOS），部分系统需要额外授权。
